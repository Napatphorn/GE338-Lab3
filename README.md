**พื้นที่ศึกษา:** บริเวณทะเลบัวแดง หนองหานกุมภวาปี จ.อุดรธานี
**เป้าหมาย:** สร้างแผนที่จำแนกการใช้ที่ดินของพื้นที่ชุ่มน้ำและเกษตรกรรมที่อยู่ติดกัน  และเปรียบเทียบประสิทธิภาพของอัลกอริทึม Machine Learning ผ่าน Google Earth Engine

จากโจทย์ที่องค์กรอนุรักษ์ต้องการแผนที่การใช้ที่ดินในพื้นที่ชุ่มน้ำและเกษตรกรรมภายใต้ข้อจำกัดเรื่อง Training Data ทางเราได้ทำการทดลองและได้ข้อสรุปดังนี้:

1. **วิธีไหนให้ผลแม่นยำที่สุดเมื่อ Training Data มีน้อย?**
   * **คำตอบ:** อัลกอริทึม **Random Forest (RF)** ให้ผลลัพธ์ที่แม่นยำและเสถียรกว่า Support Vector Machine (SVM) อย่างเห็นได้ชัด เนื่องจากลักษณะของ RF ที่สร้างต้นไม้ตัดสินใจหลายต้นช่วยลดปัญหา Overfitting เมื่อข้อมูลมีจำกัดได้ดีกว่า 
2. **ถ้าใช้แค่ Spectral Bands ปกติ vs เพิ่ม Spectral Indices เข้าไป — ต่างกันแค่ไหน?**
   * **คำตอบ:** การเพิ่ม Spectral Indices เช่น NDWI (ดัชนีน้ำ) และ NDVI (ดัชนีพืชพรรณ) ช่วยให้โมเดลมีเกณฑ์ที่ชัดเจนขึ้นในการแยกพื้นที่ชุ่มน้ำ ออกจากพื้นที่เกษตรกรรม ดัชนีเหล่านี้ทำหน้าที่เหมือนทางลัดให้โมเดลมองเห็นลักษณะทางกายภาพที่สำคัญได้รวดเร็วขึ้น
3. **แผนที่ที่ได้เชื่อถือได้แค่ไหน และในบริเวณใดบ้างที่ไม่แน่ใจ?**
   * **คำตอบ:** แผนที่นี้มีความน่าเชื่อถือในเกณฑ์ดี (Overall Accuracy 77.86%) โดยมีความน่าเชื่อถือสูงมากในการแยกแหล่งน้ำและพื้นที่เกษตรกรรมหลัก แต่จะมีความไม่แน่นอนสูงในบริเวณรอยต่อ ระหว่างพื้นที่ชุมชนกับพื้นที่เพาะปลูก และบริเวณขอบเขตของพื้นที่ชุ่มน้ำที่มีพืชน้ำขึ้นเบาบาง

## ขั้นตอนการทำงานโดยสรุป 
1. **Data Collection:** ใช้ GEE Code Editor เปิดภาพ Sentinel-2 สลับกับ False Color Composite เพื่อกำหนดจุดตัวอย่าง (Training Points) แบบ Manual จำนวน 4 คลาส 
2. **Export Data:** บันทึกจุดตัวอย่างเป็น FeatureCollection ลงใน GEE Asset
3. **Model Training:** ดึงข้อมูลจุดตัวอย่างเข้าสู่ Google Colab สร้าง Band พิเศษ (NDVI, NDWI) และสกัดค่าพิกเซล
4. **Evaluation:** แบ่งข้อมูลเป็น Train/Test (80/20) ทดสอบรันโมเดล Random Forest และ SVM ประเมินผลด้วย Confusion Matrix, ดึงค่า Feature Importance และทดลองตัดแบนด์รบกวนออกเพื่อเพิ่มความแม่นยำ

## ภารกิจที่ 1: ออกแบบ Training Strategy
**1. ประเภทที่ดินที่ใช้ (4 Classes):**
เพื่อให้ครอบคลุมบริบทของพื้นที่ทะเลบัวแดง จ.อุดรธานี ได้กำหนดคลาสการใช้ที่ดินดังนี้:
* **Class 1 (Water):** แหล่งน้ำเปิด แหล่งน้ำธรรมชาติ หรืออ่างเก็บน้ำที่ไม่มีพืชพรรณปกคลุม
* **Class 2 (Wetland):** พื้นที่ชุ่มน้ำ หนอง บึง ที่มีพืชน้ำหรือวัชพืชน้ำปกคลุม (เช่น บริเวณทะเลบัวแดง)
* **Class 3 (Built-up):** สิ่งปลูกสร้าง พื้นที่ชุมชน เมือง ถนน และสิ่งพื้นผิวที่สร้างโดยมนุษย์
* **Class 4 (Agriculture):** พื้นที่เพาะปลูก พื้นที่การเกษตร (นาข้าว พืชไร่) โดยรอบพื้นที่ชุ่มน้ำ
สาเหตุที่ไม่มี class forest เพราะว่าพื้นที่ส่วนใหญ่ในบริเวณทะเลบัวแดงไม่มีป่าไม้ในลักษณะป่าบก ส่วนใหญ่จะมีเป็นพื้นที่ wetland และ agriculture
**2. วิธีการหา Training Samples:**
เนื่องจากไม่สามารถลงพื้นที่จริงได้ จึงใช้วิธี Manual Point Sampling ผ่านการแปลตีความด้วยสายตา บนแพลตฟอร์ม Google Earth Engine โดยใช้ภาพถ่ายดาวเทียม Sentinel-2 ร่วมกับการแสดงภาพแบบ False Color Composite (B8, B4, B3) เพื่อเน้นให้พืชน้ำและพื้นที่เกษตรมีสีสันที่แตกต่างกันอย่างชัดเจน ช่วยให้จุดข้อมูลได้แม่นยำขึ้น
**3. การแบ่ง Train/Validation:**
เลือกใช้การแบ่งข้อมูลแบบ **80% Train / 20% Validation แบบสุ่ม (Random Split)** **ข้อเปรียบเทียบ:** การแบ่งแบบสุ่ม 80/20 ทำได้รวดเร็วแต่อาจทำให้เกิด *Spatial Autocorrelation* (จุดสอนและจุดทดสอบอยู่ใกล้กันเกินไป) ทำให้ค่าความแม่นยำสูงเกินจริง เพราะโมเดลเรียนรู้จากพิกเซลที่สภาพแวดล้อมคล้ายกัน ในขณะที่ **Spatial Cross-validation** จะบังคับให้โมเดลไปทดสอบในพื้นที่ย่อย (Spatial Blocks) ที่มันไม่เคยเห็นมาก่อน ซึ่งสะท้อนความสามารถที่แท้จริงของโมเดลได้ดีกว่า
**4. Feature ที่นำมาใช้:**
* **Spectral Bands:** ภาพ Sentinel-2 SR (B3, B4, B8, B11, B12) 
* **Spectral Indices:** เพิ่ม **NDVI** และ **NDWI** * **เหตุผล:** พื้นที่ศึกษาเป็นพื้นที่ชุ่มน้ำและเกษตรกรรม การใช้คลื่นอินฟราเรดคลื่นสั้น (SWIR - B11, B12) ร่วมกับ NDWI สำคัญมากในการตรวจจับความชื้นและน้ำ ส่วน NDVI จำเป็นในการแยกความหนาแน่นของพืชพรรณ

## ภารกิจที่ 2: การเปรียบเทียบอัลกอริทึม
ได้ทำการทดลองเปรียบเทียบระหว่าง **Random Forest (RF)** (ตั้งค่า 50 Trees) และ **Support Vector Machine (SVM)** โดยใช้ชุดข้อมูลและ Features ชุดเดียวกัน (หลังจากการทดลองตัด Band 2 ออก) ได้ผลลัพธ์ดังนี้:

| Metric | Random Forest | Support Vector Machine |
| :--- | :---: | :---: |
| **Overall Accuracy** | **77.86%** | 66.43% |
| **Kappa Coefficient** | **0.685** | 0.542 |
| **F1-Score (Water)** | 0.857 | 0.782 |
| **F1-Score (Wetland)** | 0.545 | 0.667 |
| **F1-Score (Built-up)** | 0.795 | 0.825 |
| **F1-Score (Agriculture)** | 0.734 | 0.648 |

*(หมายเหตุ: นำผล PA และ UA มาคำนวณ F1-Score ต่อคลาสใน Python)*

**สรุปผลเปรียบเทียบ:** Random Forest มีประสิทธิภาพโดยรวมสูงกว่า SVM อย่างชัดเจนเมื่อทำการคัดเลือก Feature ที่เหมาะสม (Feature Selection) อัลกอริทึมแบบต้นไม้ตัดสินใจสามารถหาเงื่อนไขตัดแบ่งข้อมูลที่มีความซับซ้อนในพื้นที่ชุ่มน้ำได้ดีกว่า

## ภารกิจที่ 3: Feature Importance และการตัดสินใจบนฐานข้อมูล
จากการดึงค่าความสำคัญของ Feature (Feature Importance) จากโมเดล Random Forest พบว่า:

1. **ตัวแปรที่สำคัญที่สุด:** แบนด์ **B11 (SWIR) และ B12 (SWIR)** มีความสำคัญสูงสุดในการตัดสินใจ ตามด้วย **NDWI** * คลื่น SWIR และ NDWI มีความไวต่อการดูดกลืนแสงของน้ำและความชื้นสูงมาก จึงมีประสิทธิภาพในการแยกพื้นที่ชุ่มน้ำ ออกจากแหล่งน้ำ และ เกษตรกรรม
2. **การทดลองลดมิติข้อมูล (Feature Elimination):** เมื่อสังเกตเห็นว่าแบนด์ B2 (Blue) มีความสำคัญต่ำ จึงได้ทดลอง **ตัด B2 ทิ้ง** และรันโมเดลใหม่ ผลปรากฏว่า:
   * **Random Forest:** ความแม่นยำรวมพุ่งสูงขึ้นจาก 68.38% เป็น **77.86%** เนื่องจาก B2 เป็นช่วงคลื่นสั้นที่ถูกรบกวนจากการกระเจิงในชั้นบรรยากาศ (Atmospheric Scattering) ได้ง่าย การตัดออกจึงเป็นการลด Noise ให้กับโมเดล
   * **SVM:** ความแม่นยำกลับลดลงเหลือ 66.43% แสดงให้เห็นว่า SVM อ่อนไหวต่อการเปลี่ยนแปลงจำนวนมิติของข้อมูล (Hyperplane) มากกว่า
